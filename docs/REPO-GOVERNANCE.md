# Repo Governance

## Ownership

- The repository name and primary branding are controlled by maintainers.
- No automated process may rename branding, owner references, or mission statements.
- Changes to brand files require explicit maintainer review.

## Protected areas

- `brand/`
- `docs/`
- release workflows
- GitHub ownership and repository metadata

## Contribution rule

Contributions should preserve the core product framing:

- browser workspaces
- sub-profile isolation
- automation-first design
- deterministic replay and auditability

## Release discipline

- Prefer small, reviewable commits
- Keep architecture decisions documented
- Do not merge broad branding changes without review
